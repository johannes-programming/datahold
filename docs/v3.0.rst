Overview
--------

This project wraps common mutable datastructures for inheritance with modification.

Installation
------------

To install ``datahold``, you can use ``pip``.
Open your terminal and run:

.. code-block:: shell

    pip install datahold

``__contains__(key: object, /) -> bool``
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

This magic method implements ``key in self``. It works as follows:

.. code-block:: python

    def __contains__(self: Self, key: object, /) -> bool:
        "True if the dictionary has the specified key, else False."
        return dict(self.data).__contains__(key)

The docsting is somewhat imprecise because it mimics the original method of ``dict``.

``fromkeys(iterable: Iterable[Key], value: Any = None, /) -> Self``
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

``fromkeys`` is the only ``classmethod`` of ``dict``
and therefore also of ``BaseDataDict``.
It works as follows:

.. code-block:: python

    @classmethod
    @setdoc.setdoc(dict.fromkeys.__doc__)
    def fromkeys(
        cls: type[Self], iterable: Iterable[Key], value: Any = None, /
    ) -> Self:
        return cls(dict.fromkeys(iterable, value))

See ``BaseDataDict`` and ``__contains__`` for more information.

``class datahold.base.BaseDataList.BaseDataList[Item](data: Iterable[Item] = (), /)``
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

This abstract base class inherits from ``BaseDataObject``.
It works analogously to ``BaseDataDict[Key, Value]``.
It data property is annotated as ``tuple[Item, ...]``.
It implements the methods ``__add__``, ``__contains__``, ``__getitem__``,
``__iter__``, ``__len__``, ``__mul__``, ``__repr__``, ``__reversed__``, ``__rmul__``,
``count``, and ``index`` accordingly.
It has no classmethod to implement.

``__eq__(other: object) -> Any``
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

This magic method implements ``self == other``.
Here, it establishes a standard for all its descendents.
It should only ever be overwritten with an extensional equivalent.

.. code-block:: python

    def __eq__(self: Self, other: object) -> Any:
        "This magic method implements self==other."
        if isinstance(other, BaseDataObject):
            return self.data.__eq__(other.data)
        else:
            return NotImplemented

``class datahold.DataDict[Key, Value](data: Any = (), /, **kwargs: Value)``
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

This abstract base class inherits
from DataObject, ``BaseDataDict[Key, Value]``, and ``collections.abc.MutableMapping[Key, Value]``.
It implements all methods and classmethods of ``dict``
that can change the underlying instance
(and that are not already present in ``object``).
The implementation of ``__contains__`` works like this:

.. code-block:: python

    def __contains__(self: Self, key: Any, /) -> Any:
        "True if the dictionary has the specified key, else False."
        return dict[Key, Value](self.data).__contains__(key)

Docsting and signature are somewhat imprecise because they mimic the original method of ``dict``.
The other methods -
``__contains__``, ``__getitem__``, ``__iter__``, ``__len__``,
``__or__``, ``__repr__``, ``__reversed__``, ``__ror__``,
``get``, ``items``, ``keys``, and ``values`` - work analogously.
``fromkeys`` - the only ``classmethod`` - works as follows:

.. code-block:: python

    @classmethod
    def fromkeys(cls: type, iterable: Any, value: Any = None, /) -> Any:
        "Create a new dictionary with keys from iterable and values set to value."
        return dict[Key, Value].fromkeys(*args, **kwargs)
