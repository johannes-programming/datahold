Overview
--------

This project wraps common mutable data structures for inheritance with modification.

Of course, similar base classes are provided by ``collections``.
They were also the main inspiration for this project.
But they suffer from one principal problem:
They necessitate overriding multiple methods
to effectively customize the data holder.
This project allows creating a customized data holder
by only overwriting the ``data`` property.
All proper methods and magic methods in the classes of this project
use the getter (and in the mutable ``core`` classes also the setter) of the ``data`` property.
Therefore, overwriting the ``data`` property alone immediately gives an effectively modified class.
Further overwriting of methods is still possible
and might - depending on the situation - substantially improve performance,
but it is not strictly necessary.
For example, it is possible to create - with only a few lines of code - a ``MutableSet``
that forces all its items to become strings.

.. code-block:: python

    from datahold import HoldSet

    class StrSet(HoldSet):

        __slots__ = () # all classes with Hold have a predefined slot layout with only _data

        @property
        def data(self):
            return self._data

        @data.setter
        def data(self, value):
            self._data = frozenset(map(str, value))

Installation
------------

To install ``datahold``, you can use ``pip``.
Open your terminal and run:

.. code-block:: shell

    pip install datahold

Typing
------

The typing conforms strictly to ``mypy``.

Features
--------

Inherited features are not mentioned again, unless something changed.

Classes that can be imported from ``datahold``, from ``collections.abc``, or from ``typing`` are usually mentions without import.

The same is true for protocols with self-explanatory name (i.e. ``SupportsKeysAndGetitem``),
for nested classes (i.e. ``Data``),
and for all other classes whose origin or nature should be apparent to a programmer.

    @setdoc.basic
    def __contains__(self: Self, other: object, /) -> bool:
        try:
            return other in self.data
        except TypeError:
            return other in (x for x in self.data)  # type: ignore[operator]

    @overload
    @setdoc.basic
    def __getitem__(self: Self, index: SupportsIndex, /) -> Item: ...

    @overload
    @setdoc.basic
    def __getitem__(self: Self, index: slice[Optional[SupportsIndex], Optional[SupportsIndex], Optional[SupportsIndex]], /) -> Self: ...

    @setdoc.basic
    def __getitem__(
        self: Self, index: SupportsIndex | slice[Optional[SupportsIndex], Optional[SupportsIndex], Optional[SupportsIndex]], /
    ) -> Item | Self:
        if isinstance(index, SupportsIndex):
            return self.data[index]
        else:
            return type(self)(self.data[index])
